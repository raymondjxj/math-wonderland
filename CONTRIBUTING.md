# 贡献指南 Contributing

感谢你愿意为孩子们的数学教育出一份力！

## 最快的贡献方式

- **试玩并提 issue**：哪怕只是"我家孩子卡在这一幕了"，都是宝贵的反馈
- **审校课程内容**：特别是教师朋友——数学准确性、儿童语言、难度梯度
- **写一个新单元**：纯数据文件，不需要懂框架

## 开发约定

1. **零依赖原则**：本项目刻意不使用任何框架、构建工具与第三方库。新增代码必须是原生 HTML/CSS/JS（非 module），`file://` 协议双击 index.html 必须始终可用。
2. 代码风格与现有文件保持一致（ES5 风格、2 空格缩进、`MW` 命名空间）。
3. 提交前必过：
   ```bash
   for f in $(find . -name "*.js"); do node --check "$f"; done
   node tools/validate.js
   ```
4. 课程内容（content/）遵循 [content/README.md](content/README.md) 的数据格式与写作规范：
   - 七幕结构、分层题、温故题、一题多解、"为什么"题
   - 数学内容必须准确，答案逐一验算，干扰项不撞等值
   - 儿童语言：短句、有画面感、成长型思维（夸过程不夸聪明）

## 内容审校流程

1. 课程内容 PR 需要 **1 名数学审校 + 1 名儿童语言评审**（可由同一人兼任两角色时从简）。
2. 审校清单：知识点对齐人教版？答案全对？干扰项无等值？温故题确实是旧知识？
3. 修改引擎（assets/）的 PR 需附截图或录屏说明效果。

## 行为准则

请阅读 [CODE_OF_CONDUCT.md](CODE_OF_CONDUCT.md)。这里首先是给孩子做产品的社区——耐心、善意、对新手友好。
